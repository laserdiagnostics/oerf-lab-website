## 🟡 三条优化建议

### 1. 第三页午餐场景是人最多的一格，AI 容易翻车

第三页上方主格里林澄、周岚加上两三个背景成员，总共四到五人同时出现在一个画面中，还要自然互动。这是整套漫画里最复杂的一个分镜。AI 画多人物群像时，常见问题包括：手部变形、人物之间比例失调、表情诡异、或者干脆有人融合在一起。

**建议**：

在 prompt 里把构图约束得更死。比如明确要求前景是林澄和周岚面对面坐，背景是两三个人在远处模糊处理的剪影。不要要求「围桌吃饭」这种需要所有人脸部清晰可见的高难度构图。林澄和周岚的表情是这一格的核心，背景成员只需要存在感，不需要细节。

### 2. 第二页的夜场，海军蓝在暗光下会消失

林澄的角色色是海军蓝 `#0A2540`，跟黑色非常接近。第二页第三格设定在夜晚实验室（窗外深蓝，室内暖灯），林澄的海军蓝外套在这种低照度环境下可能和背景融为一体。

**建议**：

在第二页第三格的 prompt 中加一条：让林澄的深色外套被室内暖光打出一层边缘光，或者让她在这个场景里脱掉外套露出奶油白上衣。不需要改角色设定，只需在暗光场景的单个 prompt 里做一个局部的服装调整。

### 3. 图片外 HTML 对白和图片之间的视觉衔接

目前方案是每张图片下面放对应的 HTML 对白，图片内部完全无字。这个策略本身没问题，但有一个用户体验上的小坑：读者看图片的时候不知道角色在说什么，看完图片滑到下面才看到对白，然后要回头再看一遍图片。这个「来回跳」在移动端尤其明显，因为图片和对白不在同一屏。

**建议**：

在网页设计上考虑两种方案来缓解：

- **方案 A**：对白放在图片上方（不是下方），作为阅读的引导。读者先看到「我之前没做过实验。」→ 再看图里两个人站在白板前 → 读者自动在脑补对白。
- **方案 B**：对白和图片在同一张卡片内，对白紧贴图片，中间不隔其他元素。保证在手机屏幕上至少能看到图片上半部分 + 对白行。

这两种都比「先图后字」的阅读顺序更顺。具体实现留给网页阶段，目前只需要在方案 note 里标记一下。
