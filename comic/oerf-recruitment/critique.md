# OERF 招生漫画方案 — 批评与修改建议

## 总体评价

方案的信息分层策略（漫画做情绪 + HTML 做信息）很对。分镜节奏、四个能力符号的视觉化、两页闭环结构，都是亮点。但有一个根子上的问题贯穿始终，下面按严重程度逐条说。

---

## 🔴 P0 — 必须改

### 1. 「陈老师」这个虚构人物是个炸弹

方案说「虚构合成导师，不对应某位真实教师」。想法是好的，怕画不像、怕肖像权、怕太正式。但实际效果会怎样？

一个学生看了漫画，觉得「陈老师」很亲切，发邮件说「陈老师您好，我想申请您的课题组」。收到邮件的是蔡老师。你说蔡老师该回什么？「不好意思，陈老师是编的」？

更麻烦的是，你用了虚构导师，等于在招生漫画里**主动抹掉了真实团队的面孔**。读者走进实验室发现领头的不是漫画里那个人，第一反应不是惊喜，是困惑。这种认知落差在招生场景里是致命的。

**建议**：两种改法，选一个。

- **方案 A（推荐）**：删掉陈老师，让周岚和方启（学生）当接待者。第一页是学生之间的平等欢迎，第二页是团队协作。完全不需要虚构一个不存在的导师。学生带学生，更真实也更打动人。
- **方案 B**：保留导师角色但不给姓氏，只叫「老师」，且面部特征做风格化处理（戴护目镜/侧脸/背影），不追求识别具体是谁，只追求「有个指导者在场」的氛围。

### 2. 8 个分格塞了太多东西，信息过载

你数一下每格要承载的信息量：

- P1-G3：四人围白板，公式 + 光谱 + 火焰 + 流场 + 四个符号首次出现 + 两句对白
- P2-G1：四个符号依次点亮 + 三个角色围观 + 两句长对白
- P2-G2：五阶段路径墙 + 六学科图标 + 一句长对白

AI 画漫画不是人画漫画。人画可以取舍，AI 你给多少它画多少，结果就是每格都密密麻麻跟说明书似的。招生漫画的核心是**让人想加入**，不是让人研读。

**建议**：

- 每格只放一个核心信息点。多余的砍掉。
- P2-G2 的「成长路径墙」删掉，那段信息放到 HTML 文字里。漫画里用一个更简单的画面替代：比如周岚在带林澄调光路，旁边是方启在跑模拟，三个人各干各的但在一间屋子里。这个画面比路径图更能传达「不同阶段的人都在这里做实事」。
- P1-G3 的四符号首次出现，改成只出现两个（棱镜 + 齿轮），另外两个留给第二页，分散视觉负载。

### 3. 四符号系统太隐晦，读者大概率看不懂

棱镜 = 想象力、齿轮 = 执行力、透镜拼图 = 解决问题、蓝焰 = 逆风翻盘。这个映射关系你自己觉得很妙，但一个路人读者在 30 秒内扫完两页漫画，能解码出几层？

大概率是零层。他会觉得「哦，画得挺好看的」，然后完全没意识到这四个东西跟招生的四项能力有关。

**建议**：

- 符号可以保留，但必须在漫画里**明说一次**。比如 P2-G1 里，陈老师/周岚说「想象力、执行力、解决问题、逆风翻盘」的时候，四个符号在画面中**依次出现并对齐**到对应文字。不是「隐约浮现」，是「明确对应」。
- 或者更干脆：不要符号系统。直接用四个小场景（一个人在白板上天马行空画草图 = 想象力，一个人在实验台前连续调试三天 = 执行力，两个人一起围着一个出问题的装置讨论 = 解决能力，一个人在深夜实验室重新搭崩掉的装置 = 逆风翻盘）。场景比符号好懂一百倍。

## 🟡 P1 — 强烈建议改

### 4. 「门打开以后」这个叙事起点有点老套

门外犹豫 → 门打开 → 被欢迎 → 走进去。这是十年前大学招生宣传片的标配。不是说不能用，但用在漫画里显得缺乏想象力。你们招的是「有想象力的学生」，结果自己的招生漫画开头是最没想象力的套路。

**建议**：从中间切入。第一格直接画林澄已经在实验室里，正在白板上画一个她搞不懂的问题，旁边周岚在帮她拆解。用一个画外音或标题交代「这是林澄来实验室的第一周」。省略犹豫和敲门，直接从「已经在做了」开始。这样更可信，也更像真实实验室的日常。

### 5. P2-G3 国际协作那格风险很高

方案写的是「林澄与团队面对视频协作屏幕，屏幕中为虚构的国际教授形象」。AI 画一个视频屏幕里的另一个人，效果大概率是恐怖谷。而且「虚构的国际教授」画出来要么像鬼影，要么像某真实人物，两边都是坑。

**建议**：不画视频屏幕。改成林澄和方启在看一篇刚接收的论文（屏幕上是论文标题和接收邮件），旁边墙上有一张世界地图，上面标了几个光点。一个旁白框写「高质量成果会打开更大的世界」。不画人，只画物和氛围，反而更有余味。

### 6. P2-G4 结尾画面太满了

方案要求结尾格同时出现：林澄展示结果、蓝焰稳定燃烧、团队成员围观、四个符号全回收、多色光汇聚、门打开、门外新学生。一格里塞七个视觉元素，AI 会画出一锅粥。

**建议**：结尾只保留三个东西：林澄站在实验台前展示一组漂亮的数据图（屏幕上的曲线或图像），蓝焰在装置中稳定燃烧，门开着，光线从门照进来。删掉所有符号回收和新学生剪影。结尾的力量来自克制，不是堆砌。

## 🟠 P2 — 建议优化

### 7. 人物数量偏多，两页撑不起四个人

林澄、陈老师、周岚、方启，四个角色在八格里各分不到两格。方启和周岚作为「实验」和「计算」的代表，区分度在分镜里很难体现。读者分不出谁是谁。

**建议**：如果删掉陈老师（见第 1 条），就只剩三个角色，刚好。林澄是主角，周岚是主要互动对象，方启在关键格出现一两次。方启甚至可以跟周岚合成一个人——一个既做实验又跑计算的博士生，更真实（现在哪个博士生不跨方向）。

### 8. 角色设定里「陈老师」的性别和年龄暗示

方案设定陈老师为 40 岁左右中国男性。如果在方案 A 里保留导师形象，建议模糊化：不标注年龄和性别特征，用背影/侧脸/护目镜遮挡，让读者自行代入他们未来会遇到的任何导师。

### 9. 「概念符号映射」里蓝焰的含义跟实际实验有关联但不够直接

蓝焰映射「逆风翻盘」，这个联想来自你们实际做燃烧实验的场景。但对路人读者来说，蓝焰就是蓝焰，跟逆风翻盘之间没有直观联系。用深夜实验室一个人重新搭装置的画面，比蓝焰更能传达「逆风」的感觉。

### 10. 角色名字「林澄」可以保留但想想是否有更好的

林澄（澄 = 清澈），名字不错，但跟「凌晨」谐音，可能会被读岔。如果要用，建议在漫画里不出现文字标注姓名（反正 AI 也画不好中文），名字只是给你们自己内部沟通用的。

## 🔵 P3 — 实施层面

### 11. AI 画漫画的硬伤：角色一致性、文字可读性、科学装置准确性

- **角色一致性**：分镜之间的角色长相、服装、发型很难保持一致。建议角色设定尽量简化（比如林澄永远穿海军蓝外套 + 眼镜，周岚永远白色实验服 + 低马尾），减少变化带来的不一致风险。
- **文字可读性**：AI 生成的对白气泡里的中文大概率是乱码。建议所有对白不写进气泡，而是用后期在 HTML 上叠加文字层的方式处理。或者生成图片后用空白气泡预留位置，文字在网页端用 CSS 覆盖。
- **科学装置**：光学平台、激光光路、燃烧装置等专业设备，AI 可能画得似是而非。建议在 prompt 里描述时尽量用通俗类比，不追求绝对准确。如果一张图里的装置明显画错了反而减分。

### 12. 两页 4:3 在手机上的阅读体验

方案说「桌面端左右排列，移动端上下排列」。两页 4:3 在手机上上下排列，加起来相当于一个很长的竖图，用户要滑很久。而且标准分镜的阅读顺序（右上→左下→右下）在单列竖排时会乱。

**建议**：考虑改成四页 3:4 竖版（手机原生比例），桌面端用 CSS grid 排成 2×2。或者更简单：做成一个横版长条漫（webtoon 风格），在网页上用 CSS scroll-snap 横向滑动，手机上直接竖滑。

### 13. 缺少「氛围」场景

你特别强调了「团队融洽的氛围」，但分镜里几乎全是工作场景：讨论白板、做实验、看屏幕。没有吃饭、喝咖啡、闲聊、一起吐槽实验失败、或者一起庆祝论文接收的画面。

**建议**：在某一格里塞一个非工作场景。比如 P2-G2 不用路径墙，改成实验室的公共区，几个人围在一起吃盒饭，白板上画着当天讨论遗留的问题。或者实验台上除了仪器，放一杯没喝完的奶茶。这些细节比任何文字都更能传达融洽。

### 14. 语言风格可以更活泼

分镜里的对白偏文雅：「这里没有唯一的起点」「问题可以一起拆，方法可以一起学」。这是 PPT 语言，不是真实实验室里会说的话。

**建议**：

> 「别站着了，进来吧。你本科学什么的？」（周岚）
> 「我之前没做过实验。」（林澄）
> 「巧了，我博一也不会。你看我现在不是还在做吗。」（周岚指指自己的实验台）

这才像真的。招生漫画的目的不是展示「我们有多专业」，是让读者觉得「我想跟这群人一起干活」。

---

## 总结：P0 必改清单

| # | 问题 | 修改方向 |
|---|------|---------|
| 1 | 虚构「陈老师」会引发认知错位 | 删掉或用匿名化导师/纯学生互动 |
| 2 | 8 格信息密度过高 | 每格只放一个信息点，砍掉路径墙 |
| 3 | 四符号系统读者无法解码 | 改为四个小场景，或让符号与文字明确对齐 |

P0 三条改完，方案就可以进入生成阶段。P1 和 P2 的建议能显著提升成片质量，但即使不改也不致命。
