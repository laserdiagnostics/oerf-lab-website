# OERF 招生漫画：ChatGPT 网页版生图指南

## 为什么改用网页版

Codex 内置图像服务连续两次出现网络连接错误，提示词本身没有报错。继续重试意义不大，因此改用 ChatGPT 网页版生成，再把最终 PNG 文件交回 Codex 检查和嵌入网站。

## 总体顺序

严格按以下顺序生成：

1. 生成角色参考图 `characters.png`，比例 4:3。
2. 审核角色参考图；不合格时先修正，不要继续生成页面。
3. 使用角色参考图依次生成四张漫画页，比例全部为 3:4。
4. 下载并按规定文件名保存。
5. 将五张 PNG 发回 Codex，进行画面检查、网页排版和部署。

## 第一步：生成角色参考图

1. 打开 [ChatGPT](https://chatgpt.com/)，新建一个普通对话。
2. 可以直接要求创建图片；如果界面显示工具菜单，也可以选择 `More / 更多` → `Images / 图片`。
3. 打开本地文件：
   `C:\Users\1\oerf-lab-website\comic\oerf-recruitment\characters\characters.md`
4. 复制其中 `角色参考图完整生成提示词` 标题下面的全部内容。
5. 在提示词最前面加上：

   > 请创建一张全新的角色参考图，不要生成任何文字。画幅为横向 4:3。严格按照下面的提示词生成：

6. 发送并等待生成。
7. 下载图片，保存为：
   `C:\Users\1\oerf-lab-website\comic\oerf-recruitment\characters\characters.png`

### 角色参考图验收

必须同时满足：

- 只有两套人物身份：左侧短发眼镜学生，右侧低马尾博士生。
- 左侧所有视图是同一个人，右侧所有视图也是同一个人。
- 海军蓝与暖橙服装没有串色。
- 没有姓名、标题、伪汉字、伪字母、水印或徽标。
- 头顶、手、腿和鞋没有被裁掉，手指数目正常。

如果只有一个局部问题，在同一对话中进行单点修改，例如：

> 保持人物身份、服装、构图和其他所有内容不变。只修正左侧角色不同视图中的眼镜，使其全部为相同的细边方形眼镜。仍然不要任何文字。

如果两个人物已经混在一起或整张结构错误，直接重新生成，不要连续修补。

## 第二步：生成四张漫画页

建议继续使用同一个对话，但每一页都重新上传 `characters.png`，明确它只是人物身份参考。一次只生成一页，不要把四份提示词同时发送。

每一页的操作相同：

1. 通过附件按钮上传 `characters.png`。
2. 打开对应页面提示词文件，复制全部内容。
3. 在页面提示词前加上以下固定前缀：

   > 附件 Image 1 是人物身份与服装参考，不是要修改的图片。请创建一张全新的漫画页，不要复制角色设定表的排版。严格保持两位人物的脸型、发型、眼镜和识别色。画幅为竖向 3:4。图片中不得出现任何文字、伪文字、气泡、标签、徽标或水印。下面是本页完整提示词：

4. 粘贴页面提示词并发送。
5. 检查后下载，保存为指定文件名。

## 页面验收重点

### 第 1 页

- 上下两格仍然是一张完整漫画页。
- 只有两位清晰人物。
- 白板只有线条、圆点和箭头，没有伪文字。
- 下格手部没有融合或多指。

### 第 2 页

- 三格人物身份一致。
- 光学支架带小旋钮，不是家用镜框。
- 夜场是实验台工作灯，不是家庭台灯。
- 火焰装置为透明圆柱罩、金属灰底座、小型蓝焰。

### 第 3 页

- 前景只清晰显示两位主角，背景人员必须虚化。
- 筷子平放，不拿筷子指图。
- 下格不击掌、不握手，双手各自放在桌面。
- 屏幕和世界地图没有任何文字或真实校徽。

### 第 4 页

- 中景、轻微仰视、两人全身完整可见。
- 只突出讲结果、稳定蓝焰、开放的门。
- 火焰装置与第 2 页外观一致。
- 不出现世界地图、新学生剪影、能力符号或密集群像。

## 修改原则

- 一次只改一个问题，并重复“其他内容保持不变”。
- 人物身份错误时重新上传 `characters.png` 并再次声明它是身份参考。
- 画面出现文字或伪文字时，要求重新生成无字版本，不使用软件覆盖修补。
- 不要让 ChatGPT 把角色参考图本身改成漫画场景；必须明确“附件只作参考，创建全新图片”。

## 生成完成后

把 `characters.png` 和四张页面 PNG 发给 Codex。Codex 将检查：

- 图片尺寸与画幅；
- 角色跨页一致性；
- 是否含文字、伪文字或水印；
- 手部、人物数量和科学装置是否明显错误；
- 是否适合主页桌面 2×2、移动端单列排版。

检查通过后再嵌入网站。
