# Bilingual Site Maintenance

The English site keeps the existing unprefixed URLs. The complete Chinese site lives under `/zh/`. Both languages render the same shared page views and data structures.

## Core rules

- Use `Locale`, `Bilingual<T>`, and `localize()` from `src/i18n/types.ts`.
- Put short interface labels in `src/i18n/ui.ts` and page-level copy in `src/i18n/pages.ts`.
- Keep shared route paths in `src/i18n/routes.ts`; do not concatenate the GitHub Pages base or `/zh/` manually.
- Each localized data object must contain both `en` and `zh`. `npm run check:i18n` rejects missing or structurally mismatched content.
- The language switch displays the target language: `中文` on English pages and `EN` on Chinese pages.
- Academic publication titles, authors, journals, DOI values, and paper URLs remain in their original English metadata.
- Scientific images are shared between languages; localize only their alternative text unless a separate translated image is intentionally added.

## Adding or editing content

### Team member

1. Add the photo under `public/images/team/`.
2. Add one record to `src/data/members.ts`, including both `content.en` and `content.zh`.
3. Treat `Member.id` as a permanent public URL slug. The English and Chinese detail routes are generated automatically.

### Research area

Add or edit one bilingual record in `src/data/research.ts`. Keep the shared image path and dimensions outside the localized `content` object.

### Featured highlight

Add its publication to `src/data/publications.ts`, then add bilingual presentation copy to `src/data/highlights.ts`. The highlight ID must match the publication ID.

### UI or page copy

Add the same typed key to both languages in `src/i18n/ui.ts` or `src/i18n/pages.ts`. Do not scatter `locale === "zh"` text branches through components.

## SEO and fonts

- `BaseLayout.astro` owns canonical URLs, `hreflang`, Open Graph metadata, JSON-LD, and the default shared OG image.
- `ogImage` is an optional page override. When omitted, `public/images/og/oerf-lab-share.webp` is used.
- `@fontsource-variable/inter` supplies one self-hosted Latin variable-font file. Chinese text uses the operating-system Chinese font stack. Do not restore Google Fonts runtime requests.
- The development server is configured with the isolated `.astro-dev/` output location; production builds write to a freshly cleaned `dist/`. This prevents local development from deleting sitemap files or leaving development manifests in the deployment artifact.

## Required verification

Run before every deployment:

```bash
npm run verify
```

This checks Astro types, bilingual data parity, the production build, route pairs, language-switch intent, canonical/hreflang/OG metadata, sitemap alternates, font preload, publication counts, stale manifests, and all internal resources.
